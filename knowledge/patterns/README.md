# Patterns

Patterns are reusable operating methods extracted from completed work.

## Subdirectories

- `emerging/`
  - useful, repeated, but still settling
- `reusable/`
  - recommended by default, but not yet formalized as skills

Within each stage, store patterns under domain buckets such as `browser/`, `desktop/`, `marketplaces/`, `platform/`, and `general/`.

## Promotion Rule

Patterns may be upgraded automatically within `knowledge/`.

Formal promotion into `skills/` still requires explicit user approval.

## Current Default

At the moment, the repository's validated reusable patterns live under `reusable/`.
