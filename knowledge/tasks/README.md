# Task Records

This directory stores completed task records.

## Organization

- `browser/`
  - Browser-centric tasks and web verification work.
- `desktop/`
  - Desktop automation and WPS-heavy initiatives.
- `marketplaces/`
  - Marketplace research and report-generation tasks.
- `platform/`
  - Internal capability-building initiatives that changed the platform itself.

## Filing Rules

- One file should describe one finished initiative or one tightly related task family.
- A task record should point to scripts, outputs, artifacts, and follow-on knowledge.
- If a task only produced artifacts and no reusable insight, it does not need a long record.
- If several small task variants share the same workflow and lessons, group them into one family record.
