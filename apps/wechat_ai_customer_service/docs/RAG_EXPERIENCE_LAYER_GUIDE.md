# RAG 经验层设计说明

## 目标

RAG 经验层用于记录客服在“非标准结构知识”场景中成功回复过的经验概括。它解决的是“这类问题以后可以参考之前怎么答”的问题，但不替代正式知识库。

## 基本原则

1. RAG 经验默认采纳为经验参考。
2. RAG 经验不写入正式结构化知识库。
3. 用户可以在管理台的“RAG经验”页面一键废弃不合适的经验。
4. 被废弃的经验保留审计痕迹，但不再出现在默认采纳列表，也不再参与 RAG 检索。
5. 需要长期稳定生效的业务规则，必须通过“知识生成器”或“候选审核”进入正式知识库。

## 存储位置

默认租户的 RAG 经验存放在：

```text
apps/wechat_ai_customer_service/data/tenants/default/rag_experience/experiences.json
```

每条经验包含：

- 客户问法
- 实际回复要点
- 命中的 RAG 资料片段
- 意图识别结果
- 安全判断摘要
- 使用次数
- 当前状态：`active` 或 `discarded`

## 运行时写入条件

监听回复流程只有在以下条件都满足时才会写入 RAG 经验：

1. 本轮回复使用了 RAG 应答层。
2. 回复已成功发送。
3. 写入失败不会影响客服回复主流程。

写入后的 active 经验会作为 `rag_experience` 类型参与 RAG 索引；索引会在经验文件变化后自动重建。

## 管理台行为

新增“RAG经验”页面：

- 展示默认采纳的经验概括。
- 展示客户问法、回复要点和命中资料。
- 提供“废弃”按钮。
- 废弃后该条经验从默认列表移除。

## 与正式知识库的边界

RAG 经验层是侧写层，不具备正式业务规则效力。它可以帮助系统更像真人地理解上下文，但以下信息仍必须以正式知识库为准：

- 价格
- 优惠
- 账期
- 合同
- 赔偿
- 售后承诺
- 发票和公司主体信息

如果 RAG 经验中出现值得长期保留的规则，操作人员应把它整理为正式知识，再通过管理台审核入库。
