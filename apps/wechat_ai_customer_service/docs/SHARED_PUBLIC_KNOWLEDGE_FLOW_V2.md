# 共享公共知识库 V2 开发文档

## 目标

共享公共知识库不再由客户侧直接写入。最终链路统一为：

1. customer 正式知识库新增或更新知识。
2. 客户端只扫描未检查过的正式知识，调用 LLM/规则判断是否具备跨客户、跨行业的公共价值。
3. 适合共享的内容提交到服务端“候选共享公共知识库”。
4. 服务端 admin 审核候选时，系统再次用 LLM/规则从通用性、重复性、风险三方面给出建议。
5. admin 手动采纳后，服务端生成正式共享公共知识和已发布补丁。
6. admin 手动推送补丁，客户端启动或周期同步时拉取命令并热更新本地共享知识。

## 现状盘点

已具备：
- 服务端已有共享候选、正式共享知识、补丁发布、补丁推送和本地节点命令轮询。
- 客户端已有共享补丁预览/应用能力。
- 服务端已有从 customer formal knowledge 提炼候选的基础接口。
- 客户端已有 VPS 注册节点、命令轮询、更新检查接口。

已优化：
- 服务端增加 `shared_scan_state`，用正式知识内容指纹记录检查状态，默认只扫描未检查内容。
- 候选生成前会同时比对既有候选和正式共享库，避免同源或正文重复。
- 服务端候选对象增加 `review_assist`，用于展示 admin 审核建议。
- admin 可点击“AI复核建议”重新调用 LLM 判断候选是否适合成为公共知识。
- customer 账号允许执行自身数据空间的命令轮询和更新检查；guest 仍禁止。

新增：
- 客户端 `/api/sync/shared/formal-candidates`：提交 customer 正式知识中未检查的公共候选。
- 客户端启动后自动执行：注册节点、轮询命令、检查更新、提交未检查正式知识候选。
- 正式知识手动新增/编辑、待确认知识应用入库后，后台触发一次共享候选扫描。

## 状态模型

`shared_scan_state` 以 formal knowledge 的 `source_key` 为键。`source_key` 包含 tenant、分类、条目 ID 和正文摘要；正文改变后会自然形成新的检查对象。

常见状态：
- `proposed`：已在服务端生成候选。
- `uploaded`：客户端已提交服务端候选。
- `not_recommended`：LLM/规则判断不适合作为共享候选。
- `already_pending_or_reviewed`：服务端已有同源候选。
- `already_in_shared_library`：正式共享库已有同源或同内容条目。
- `accepted`、`rejected`、`void`：admin 已审核候选。

## LLM 使用规则

能用 LLM 的环节默认使用 LLM，但必须保持人工确认边界：
- customer formal knowledge -> shared candidate：LLM 只负责建议，不直接写正式共享库。
- shared candidate admin review：LLM 给出建议，不自动采纳。
- LLM 失败时使用保守规则预判，页面会标识为规则预判。

共享候选的判断标准不同于 customer AI经验池：这里必须足够通用，能适用于不同客户、不同行业。带具体商品、价格、库存、车型、门店、客户承诺的内容，应拒绝或要求修改。

V2.1 收紧规则：
- 不允许把单个 customer 的行业规则、门店流程、商品政策、聊天客户资料“改写泛化”后推入共享公共知识库。
- 含客户姓名、手机号、微信号、订单、合同、门店、城市、商品、车辆、报价、库存、优惠、物流/售后承诺、内部备注的内容，必须留在该 customer 自己的正式知识库。
- LLM 如果明确返回空候选，不再落回宽松规则兜底；只有 LLM 不可用时才使用保守规则。
- 保守规则只允许人工转接、礼貌表达、隐私/敏感信息、不要承诺、人工确认等极窄的跨行业客服原则。

## 热更新机制

1. Local 客户端登录后注册本机节点。
2. Admin 在服务端采纳候选，生成 signed shared patch。
3. Admin 点击推送补丁，服务端创建 `pull_shared_patch` 命令。
4. 客户端启动和每 10 分钟周期同步时轮询命令。
5. 客户端收到补丁后先预览校验，再应用到本地共享知识目录。
6. 命令执行结果回传服务端，便于 admin 查看。

客户端也会检查 `/v1/updates/latest`，用于后续版本更新提示；共享知识补丁与程序版本更新分开处理。

## 失败保护

- 生成候选不写正式共享库。
- 客户端提交失败不标记为已完成，后续可重试。
- 已检查指纹只在 LLM/规则完成或服务端返回后写入缓存。
- 共享补丁应用前必须通过路径安全校验，禁止目录逃逸。
- 未配置 VPS 时，客户端不调用 LLM 扫描公共候选，避免无意义 token 消耗。
