# Web 管理台产品化与 AI 知识生成器改造方案

## 目标

本轮改造把 Web 管理台从“开发者可用”升级为“普通业务用户可用”。正式知识仍以 `data/knowledge_bases/` 下的分类知识库为根源，不再把所有知识混在一个文件夹，也不让用户直接面对 JSON。

重点变化如下：

1. “编辑草稿”改为“AI 知识生成器”。
2. 用户用自然语言描述新增知识，系统自动判断归属门类。
3. 信息不完整时，AI 用对话方式追问关键字段。
4. 信息完整后，展示业务化总览表，用户确认后再写入正式知识库。
5. 知识库编辑页只显示业务表单，默认只读，点击“编辑”后才允许修改。
6. 一键检测显示结论、问题来源、关键细节，并支持“标记忽略”。
7. 备份/还原使用二次确认，执行前自动留存反悔快照。
8. “系统状态”合并到总览，不作为重复主页面。

## 根源结构原则

所有正式知识必须写入以下分类根目录：

- `products/items/`: 商品资料、价格、库存、物流、售后、报价模板。
- `policies/items/`: 开票、付款、物流、售后、合同、优惠、转人工规则。
- `chats/items/`: 真实客服话术、边界处理样例、闲聊风格样例。
- `erp_exports/items/`: ERP 导出、订单字段、客户资料字段映射。
- `custom/<category_id>/items/`: 用户自定义模块。

前端的新增、编辑、AI 生成、候选审核、诊断修复都必须通过分类知识库接口操作，不允许重新写回旧的统一 JSON 作为正式源。`data/compiled/structured_compat/` 只作为兼容缓存，不能作为管理台编辑源。

## AI 知识生成器流程

AI 知识生成器是旧“编辑草稿”的替代能力，不再提供裸 JSON 编辑器。

流程如下：

1. 用户在输入框里描述一条知识，例如“新增一个 200L 商用冷柜，单价 1999，10 台以上 1880，24 小时发货”。
2. 后端创建生成会话，加载分类目录、各分类 schema 和少量已有样例。
3. DeepSeek 根据描述判断门类，抽取字段，生成候选知识。
4. 后端用 schema 做硬校验，并补充业务校验。
5. 如果缺少关键字段，返回追问问题，而不是保存。
6. 用户继续回答，系统把新信息合并到会话草稿。
7. 所需字段齐全后，返回总览表、目标门类、风险提示和保存按钮状态。
8. 用户点击确认保存，系统自动生成知识 ID，写入对应分类 `items/` 目录。
9. 保存后触发兼容编译，并在总览/知识库页面立即可见。

## DeepSeek 使用边界

DeepSeek 用于理解自然语言、归类、抽取字段和生成追问。DeepSeek 不能绕过后端校验，也不能直接写文件。

后端必须始终执行：

- 分类 ID 必须存在于 registry。
- item ID 必须符合安全规则。
- 必填字段必须非空。
- 阶梯价格必须数量递增、价格递减。
- 高风险承诺必须标记转人工或风险等级。
- 保存前必须由用户点击确认。

当 DeepSeek 不可用时，系统使用确定性兜底解析，保证管理台不崩溃，并清楚提示“AI 未连接，仅完成基础识别”。

## 知识库表单体验

正式知识库页面采用“列表 + 只读详情 + 编辑模式”：

- 默认只读，避免误操作。
- 点击“新增知识”进入空表单编辑。
- 点击“编辑”后当前条目才可修改。
- 点击“取消”恢复只读。
- 点击“保存”前做客户端校验和后端校验。

字段展示规则：

- 文本字段显示普通输入框或多行文本。
- 标签字段显示逗号/换行分隔输入，不显示数组格式。
- 布尔字段显示开关/勾选。
- 单选字段显示下拉框。
- 阶梯价格显示多行表格，每行包含“数量”和“单价”，支持新增/删除档位。
- 标准回复模板显示为业务字段，如“默认回复”“议价回复”“物流回复”“售后回复”，不显示对象 JSON。
- 未识别的对象字段使用“键值对表格”编辑，不显示原始 JSON。

## 候选审核说明

候选审核用于处理“资料上传 + AI 学习”生成的待入库知识。它不是直接编辑功能，而是审核 AI 从文件中提取出的建议。

用户应该看到：

- 候选来源文件。
- 建议写入哪个门类。
- 建议新增或修改什么。
- 核心证据摘录。
- 风险提示。
- “应用入库”和“拒绝”两个动作。

应用候选前必须自动备份。拒绝候选时记录原因。

## 一键检测与忽略

一键检测输出必须业务化：

- 总体是否通过。
- 发现多少错误、警告、提示。
- 每条问题属于哪个门类或知识条目。
- 问题的关键描述。
- 可执行动作：一键修复、标记忽略。

“标记忽略”用于用户确认某个问题是可接受状态。忽略信息保存在运行时管理目录，不写入正式知识库。再次检测时默认跳过已忽略问题，但应保留后续扩展“显示已忽略”的入口。

## 备份还原

一键备份使用 `POST /api/versions`。如果前端遇到 `405 Method Not Allowed`，优先判断本地服务是否未重启到最新代码。

一键还原必须二次确认。还原前自动创建“before rollback”快照，确保用户反悔时还能恢复。

## 交付标准

本轮完成后，至少通过：

- Python 全量编译。
- 前端 JavaScript 语法检查。
- JSON 数据校验。
- admin backend 全章节检查。
- 知识库结构/运行时/编译器检查。
- 离线客服回归。
- workflow 逻辑检查。
- DeepSeek 边界 probe。
- 本地 Web 管理台桌面和移动视口 smoke。
