# RAG 与正式知识库持续优化方案

## 背景

当前系统已经把正式知识库、RAG 参考资料、RAG 对话经验分开管理。正式知识库负责确定性业务规则，RAG 负责弱召回、原始资料证据和自然表达补充。这个方向是正确的，但在进入更长期的实盘使用前，还需要补上一层“经验质量治理”：

1. 不是所有 RAG 对话经验都应继续参与检索。
2. RAG 经验和正式知识库的关系需要可解释、可追踪、可缓存。
3. 用户在管理台看到经验时，应能理解“为什么保留、为什么建议升级、为什么不参与检索”。
4. 实盘测试后发现的 RAG 表达习惯，应默认沉淀在经验层，但不能绕过候选审核直接进入正式知识库。

## 优化目标

- 给每条 RAG 对话经验生成质量评分和检索准入状态。
- 低质量、风险性、证据不足、曾触发人工确认的经验，不再参与默认 RAG 检索。
- 管理台展示每条经验的质量等级、准入状态、原因、正式知识关系和建议操作。
- 将“RAG 经验与正式知识关系”写入经验记录的缓存字段，便于后续诊断、审计和运营统计。
- 保持正式知识库最高优先级，RAG 经验只做参考和候选来源。

## 核心原则

### 1. 正式知识仍是权威层

客服回复决策顺序不变：

1. 安全规则和人工接管规则。
2. 正式结构化知识库。
3. RAG 参考资料与 RAG 对话经验。
4. 大模型表达润色。

RAG 命中不能授权价格、账期、合同、赔付、售后责任、发票税务、先发货等高风险承诺。

### 2. RAG 经验需要质量门槛

RAG 经验默认可以被记录和展示，但只有满足质量要求的 active 经验才能进入检索索引。准入标准包括：

- 有清晰客户问题和客服回复。
- 原始 RAG 命中分数达到最低阈值。
- 没有风险词。
- 没有人工接管要求。
- 推荐动作不是人工审批、转人工或拒答。
- 使用次数越多，可信度可适当提高。

### 3. 经验治理不等于正式入库

质量高的 RAG 经验可以继续作为“表达参考”和“弱召回补充”。如果它反复出现且具有业务价值，管理台只能把它升级为“待确认知识”，仍必须经过候选审核后才进入正式知识库。

### 4. 用户看到的是解释，不是内部结构

管理台展示业务语言：

- 质量高/中/低/阻断。
- 是否参与检索。
- 不参与检索的原因。
- 是否已被正式知识覆盖。
- 是否疑似和正式知识冲突。
- 是否建议升级为待确认知识。

## 数据设计

### RAG 经验新增字段

每条经验新增 `quality`：

```json
{
  "score": 0.82,
  "band": "high",
  "retrieval_allowed": true,
  "reasons": ["evidence score is strong", "no risk terms"],
  "signals": {
    "hit_score": 0.91,
    "reply_count": 2,
    "has_risk_terms": false,
    "must_handoff": false
  },
  "evaluated_at": "2026-04-29T10:00:00"
}
```

每条经验新增 `formal_relation_cache`：

```json
{
  "relation": "supports_formal",
  "recommended_action": "keep_as_supporting_expression",
  "formal_match": {
    "category_id": "product_explanations",
    "item_id": "fl-920-install",
    "similarity": 0.62,
    "title": "FL-920 安装说明"
  },
  "formal_item_count": 82,
  "evaluated_at": "2026-04-29T10:00:00"
}
```

### 检索准入规则

只有同时满足以下条件，经验才会进入 RAG 索引：

- `status == active`
- `quality.retrieval_allowed == true`
- `quality.band` 为 `high` 或 `medium`
- 没有风险词和人工接管信号

低质量经验仍保留在管理台，供用户废弃、升级、人工查看或后续改造。

## 前端设计

“对话中学到的经验”每条记录显示：

- 经验概括。
- 质量等级和分数。
- 是否参与检索。
- 质量原因。
- 客户问题、回复要点、命中资料。
- 和正式知识的关系。
- 建议动作。
- 废弃、升级为待确认知识按钮。

统计卡片新增：

- 可参与检索。
- 需观察。
- 已阻断。
- 建议升级。

## 测试策略

### 自动测试

- 高质量经验可以被检索。
- 废弃/升级经验不能被检索。
- 低分经验会记录但不会进入检索。
- 有风险词经验会记录但不会进入检索。
- 经验列表能返回质量字段和关系缓存。
- 管理台静态检查能看到质量展示元素。
- PostgreSQL 和 JSON fallback 均通过。

### 实盘测试

沿用“文件传输助手”自测：

- 普通业务问题：正式知识优先。
- RAG 软参考问题：可用时给出自然回复。
- 高风险问题：不能被 RAG 经验放行，必须转人工。
- 对话经验生成后：管理台可见质量与关系状态。

## 验收标准

- RAG 经验质量评分稳定生成。
- 低质量或高风险 RAG 经验不会进入索引。
- 管理台能解释每条经验为什么参与或不参与检索。
- 正式知识关系缓存可用，且不造成频繁无意义写入。
- 全量自动化测试通过。
- 文件传输助手实盘测试通过。
- 测试后 PostgreSQL 与 JSON 镜像状态可恢复到一致。
