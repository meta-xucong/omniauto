# 江苏车金二手车客服回复与运行态优化记录 2026-05-04

## 本轮目标

- 针对真人微信测评结果，优化模型参与方式、回答边界和候选知识生成规则。
- 在 WEB 客户端新增微信智能客服一键启动/停止能力。
- 新增自动显示的运行状态浮窗，展示空闲、思考中、已停止三种状态。

## 已落地优化

1. 运行态管理
   - 新增后台托管服务，可从 WEB 调用启动、停止、读取状态。
   - 新增托管循环脚本，循环执行微信监听任务并写入状态。
   - 自动客服工作流会在读取消息、调用 LLM、发送前后写入运行状态。

2. 浮窗与控制台
   - 微信智能客服页面新增运行状态卡片和启动/停止按钮。
   - 页面右下角新增浮窗，程序打开后即可看到状态；登录后可直接启停。
   - 状态中附带最近会话、模型层级、RAG 命中数等排障线索。

3. 回复规则
   - 增加“客户连续发消息时暂停旧回复”的保护，避免 LLM 思考期间客户补充内容后仍发送过时回答。
   - 强化私下转账、个人账户收款、发票高开/低开、贷款包过、股票建议等硬边界。
   - 保持 Flash/Pro 双模型路由，普通场景优先 Flash，高风险、权威、长上下文或 RAG-only 权威场景切 Pro。
   - 继续要求 RAG 作为一等证据参与自然回复，但禁止 RAG 单独授权价格、库存、金融、合同、车况保证等敏感承诺。

4. 检索与知识晋升
   - 商品候选检索增强口语别名和车型简称，例如“320”“宝马3系”可命中宝马 320Li。
   - RAG 经验升级过滤系统流水线文本，避免把 `RAG experience -> ... candidates=...` 这类内部记录升级成待确认知识。
   - 候选知识生成增加质量闸门，过滤“待分类规则 / unknown”、无客户问法的话术、未挂商品的商品专属知识。

5. 客户线索
   - 修复“姓名：张三 电话：...”写在一行时姓名吞掉后续字段的问题。
   - 高风险折扣与客户资料混合出现时，继续阻止自动写入 Excel，并给客户明确人工确认回复。
