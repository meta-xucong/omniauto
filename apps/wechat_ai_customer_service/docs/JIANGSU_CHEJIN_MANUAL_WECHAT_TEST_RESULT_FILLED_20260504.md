## 主要问题

1. **连续发送时存在回复错位。**  
   B05/B06、E09/E10、部分 D 类连续问题都出现过“系统回复上一条，但用户已经发了下一条”的现象。真实客户连续追问很常见，建议增加 per-conversation 消息队列、回复锁、去抖窗口，以及“如果未回复消息超过1条，优先合并理解最近一组客户消息”的机制。

2. **部分商品命中不稳。**  
   宝马320被过度转人工，说明商品 alias / 车型简称 / SKU 检索仍需加强。建议把“宝马320、320Li、3系、M运动”这类别名加入商品检索索引，并让 LLM 在低风险商品解释类问题中优先尝试商品库/RAG 命中。

3. **候选知识生成偏宽。**  
   2026-05-04 产生 117 个候选引用，其中出现多条“待分类规则”“unknown”类候选，把普通客服回复也当成政策候选。建议候选生成加更严格的新颖性、可复用性、完整性门槛；只把稳定业务规则、商品事实、可复用话术推入待确认知识。

4. **高风险合规话术还可更硬。**  
   “个人微信转定金”“少开发票”“股票建议”没有越界，但措辞仍可更清晰。建议补正式规则：个人收款必须拒绝、发票必须按实际交易、投资建议不处理且不转汽车销售。

5. **缺少逐条回复的模型路由审计。**  
   目前能看到发送日志和候选知识 LLM 辅助记录，但不能直接从日志中反查每条微信回复用了 Flash 还是 Pro、输入输出 token、是否触发 RAG、命中了哪些证据。建议给每条回复写 `trace_id / model_tier / model_name / rag_hit_ids / prompt_token / completion_token / guard_result`。

## 值得保留的正向结论

- 未确认 RAG 经验没有直接参与自动回复：2026-05-04 新增经验全部 `retrieval_allowed=false`，符合“未确认不参与回答”的设计。
- 高风险边界整体守住：贷款包过、事故/水泡/火烧保证、异地迁入、试驾安排、合同变价等都没有直接承诺。
- 上下文能力已经可用：能理解“它”“这台”“之前问过的省油代步车”等指代。
- 查无车源时没有明显编造：奥德赛、奔驰E级、坦克300、全新海鸥、5万内自动挡SUV等都能说明库中暂无明确信息并转人工。

## 建议下一步

1. 修复连续消息错位：加会话级回复锁和批处理窗口。
2. 优化商品别名检索：重点补 BMW 320/3系、车型简称、英文/中文混写。
3. 收紧候选知识生成：普通聊天记录先留 RAG 经验，只有高质量、可复用、非重复内容才进待确认知识。
4. 补强合规规则：个人定金、发票金额、投资建议。
5. 增加 LLM/RAG trace 日志，方便后续真实客户测试时判断到底是模型没想好，还是检索没命中。
