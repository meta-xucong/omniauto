# VPS-LOCAL 管理体系产品化开发需求文档

## 1. 目标

把现有微信自动客服程序改造成普通客户可理解、管理员可集中运维的双端系统：

- Local 客户端继续负责实际客服、知识库维护、RAG 资料和本地运行。
- VPS 服务端负责登录授权、客户账号管理、共享公共知识候选审核、客户数据备份、版本发布和远程运维入口。
- admin 是唯一隐藏管理员账号，可登录 VPS 服务端和任意 Local 客户端。
- customer 是每个客户自己的账号，只能管理自己的本地专业知识与 RAG 数据。
- guest 是只读授权账号，只能查看被授权范围，不能修改。

本轮开发重点是补齐上一次原型中用户难以理解或链路不完整的地方，让服务端和客户端都能按“普通用户看得懂”的方式运转。

## 2. 角色和权限

### 2.1 admin

- 唯一、通用、隐藏给客户感知之外。
- 可登录 VPS admin 控制台。
- 可登录任意 Local 客户端。
- 可查看、创建、修改、删除客户账号和授权。
- 可查看、下载、删除客户数据包。
- 可选择指定客户打包数据，而不是默认一键打包所有客户。
- 可查看、增删改共享公共知识正式库。
- 可审核 Local 上传的共享公共知识候选：采纳、作废、拒绝。
- 可登记版本更新包，并向 Local 端下发检查更新或备份/还原命令。

### 2.2 customer

- 每个客户一个 customer 账号。
- 只能管理自己的：
  - 用户商品专属知识
  - 用户正式知识
  - 用户 RAG 资料
  - 用户 RAG 经验
- 可以手动触发“把本地共享公共知识新增内容上传为候选”，但不能直接改 VPS 正式共享公共知识库。
- 可按自己的设置启用/关闭个人数据云端备份。
- 不能看到 admin 隐藏入口。

### 2.3 guest

- 由 admin 或 customer 授权。
- 只读查看授权范围内的数据。
- 不能新增、修改、删除、同步、备份、还原。

## 3. VPS admin 控制台需求

### 3.1 首页

显示系统概况：

- 客户数量、用户数量、客户电脑连接数量。
- 待审核共享知识候选数量。
- 客户数据包数量。
- 最近一次共享知识同步、最近一次客户数据备份。
- 给管理员的可执行建议，避免只有技术指标。

### 3.2 客户与权限

功能：

- 新增客户账号。
- 新增 guest 授权账号。
- 修改账号状态、显示名、角色、授权租户。
- 删除非 admin 账号。
- 不在主界面暴露“生成 test01 测试客户”按钮。

说明：

- test01 仅作为本地开发、回归测试和迁移脚本使用。
- 如果需要初始化 test01，用脚本或隐藏接口执行，避免管理员误以为这是正式业务动作。

### 3.3 客户数据

功能：

- 列表展示所有客户数据包。
- 每个数据包可查看详情：
  - 所属账号
  - 所属客户/租户
  - 数据范围
  - 文件大小
  - 创建时间
  - 数据摘要
  - manifest 文件清单
- 每个数据包可下载到管理员电脑。
- 每个数据包可删除，删除时同时清理服务端记录和对应备份文件。
- 管理员可以选择指定客户/租户生成数据包。
- 默认不提供“无差别打包全部客户”的主按钮，避免误操作和数据过量上传。

验收口径：

- 管理员能从网页上看清楚某个客户包里大概有哪些数据。
- 管理员能下载 zip 包。
- 管理员能删除误操作包。
- 打包动作必须先选择客户。

### 3.4 共享公共知识

共享公共知识分两层：

- 候选库：Local 上传的新内容，等待管理员审核。
- 正式库：管理员确认后进入 VPS 正式共享库，并成为后续发给 Local 的更新补丁来源。

正式库功能：

- 列表查看。
- 查看详情。
- 新增条目。
- 修改标题、分类、正文、状态、来源备注。
- 删除/作废条目。

候选库功能：

- 查看候选详情。
- 采纳候选：写入正式库，并生成共享知识补丁。
- 拒绝候选：保留审核记录，不进入正式库。
- 作废候选：用于误上传、重复上传或不应再展示的候选。

验收口径：

- Local 上传的新增共享知识不会直接污染正式库。
- admin 必须能看见候选内容，再决定是否采纳。
- admin 必须能直接维护正式共享公共知识。

### 3.5 客户电脑连接

原“本地端连接”改名为“客户电脑连接”。

页面解释：

- 这里显示已经接入 VPS 的客户本地程序。
- 客户电脑在线后，VPS 才能向它下发备份、还原、检查更新等命令。
- 如果没有连接记录，不代表客户数据丢失，只代表当前没有 Local 程序向 VPS 报到。

功能：

- 查看 node_id、客户/租户、在线状态、版本号、最后心跳时间。
- 查看下发命令队列。
- 给指定客户电脑下发命令时必须显示作用范围和风险提示。

### 3.6 备份与还原

功能：

- 一键要求指定客户电脑备份全部数据到 VPS。
- 一键要求指定客户电脑备份指定客户数据。
- 从 VPS 现有备份包发起还原命令。
- 还原默认 dry-run，正式覆盖需要管理员明确关闭 dry-run。

说明：

- “服务端立即打包”只适合 VPS 能访问到的数据目录，开发环境可以用于模拟。
- 真实部署中，客户电脑本地数据应由 Local 接收 VPS 命令后主动打包上传。

### 3.7 版本更新

不建议让管理员直接填写 GitHub 仓库网址后点击更新。标准流程是：

1. 开发者构建一个版本发布包，例如 zip 或安装包。
2. 把发布包放在 VPS 可下载地址，例如 `https://域名/downloads/wechat-ai-cs-1.2.0.zip`。
3. 在 VPS admin 控制台登记版本号、说明、下载地址、发布渠道。
4. Local 客户端定期或手动调用检查更新接口。
5. Local 看到新版本后，下载发布包并执行本地更新流程。

本轮只要求把 VPS 端“登记版本包”和 Local 端“检查更新”链路说明清楚，后续再把下载、校验、安装、回滚做成完整安装器。

## 4. Local 客户端需求

### 4.1 登录界面

Local 打开后先显示登录页：

- 用户名。
- 密码。
- 登录按钮。
- 登录失败提示。
- 成功后进入原客户端界面。

账号行为：

- customer/test01 登录后只看到自己的客户数据入口。
- admin 登录后看到额外的共享公共知识入口。
- guest 登录后只读。

### 4.2 专业知识库

原“知识库”继续表示当前账号自己的专业知识：

- 用户商品专属知识。
- 用户正式知识。
- 用户 RAG 资料。
- 用户 RAG 经验。

页面文案应避免让 customer 误以为这里能直接改全局共享公共知识。

### 4.3 共享公共知识库

只在 admin 登录 Local 时展示。

功能：

- 查看本地共享公共知识条目。
- 新增条目。
- 修改条目。
- 删除或作废条目。
- 手动上传新增内容到 VPS 候选库。

customer 和 guest 不展示该入口。

### 4.4 共享知识候选上传

Local 提供手动上传按钮，并为后续定时上传预留服务接口：

- 扫描本地共享公共知识。
- 计算每条内容 digest。
- 已上传过相同 digest 的条目跳过。
- 新增或变化内容上传到 VPS `/v1/shared/proposals`。
- 上传后只进入 VPS 候选库，等待 admin 审核。

## 5. 数据结构

### 5.1 客户数据包

字段：

- `package_id`
- `account_username`
- `tenant_id`
- `scope`
- `backup_id`
- `package_path`
- `bytes`
- `summary`
- `manifest`
- `created_by`
- `created_at`

新增行为：

- `GET /v1/admin/customer-data/{package_id}` 查看详情。
- `GET /v1/admin/customer-data/{package_id}/download` 下载。
- `DELETE /v1/admin/customer-data/{package_id}` 删除记录和文件。
- `POST /v1/admin/customer-data/package-customer` 按客户打包。

### 5.2 共享公共知识正式库

字段：

- `item_id`
- `category_id`
- `title`
- `content`
- `status`
- `source`
- `tenant_id`
- `created_by`
- `created_at`
- `updated_by`
- `updated_at`

新增行为：

- `GET /v1/admin/shared/library`
- `POST /v1/admin/shared/library`
- `GET /v1/admin/shared/library/{item_id}`
- `PATCH /v1/admin/shared/library/{item_id}`
- `DELETE /v1/admin/shared/library/{item_id}`

## 6. 测试策略

### 6.1 本地模拟测试

本轮优先用本地模拟完成交付：

- FastAPI TestClient 测 VPS admin API。
- FastAPI TestClient 测 Local admin API。
- Node `--check` 校验前端 JS 语法。
- 启动本地 VPS 和 Local 服务后做 HTTP 烟测。

本地模拟足以覆盖：

- 登录。
- 权限。
- 客户数据包详情、下载、删除。
- 共享知识正式库 CRUD。
- 候选上传和采纳。
- UI 关键文案和按钮存在性。

### 6.2 真实 VPS 测试

真实 VPS 适合放在交付前或上线前执行：

- VPS 绑定域名或公网 IP。
- 配置 HTTPS。
- Local 指向真实 VPS。
- test01 登录授权。
- Local 上传候选共享知识到真实 VPS。
- VPS 发起指定客户备份命令。
- Local 回传备份包。
- VPS 登记一个测试版本，Local 检查更新。

当前需求不强制必须给真实 VPS 才能开发；代码可先在本地完成闭环，再搬到真实 VPS 验证网络和部署细节。

## 8. 二轮体验修正决策

### 8.1 登录与切换账号

- VPS 服务端采用独立登录页，登录后进入 admin 控制台。
- VPS 主页面提供“退出登录”按钮。
- Local 客户端主页面提供“退出登录”按钮，退出后回到登录页，方便切换 `admin`、`test01` 或其他 customer/guest。

### 8.2 客户账号和访问权限语义

对普通管理员来说，不再暴露“租户”概念：

- `customer` 账号就是客户本人。
- 创建 customer 时，系统内部自动创建同名的数据隔离范围。
- `customer` 只能访问、修改自己的数据。
- `guest` 只能由 admin 指定可查看哪个 customer。
- `guest` 不能修改、同步、备份或还原。

后端仍保留 tenant 作为内部隔离边界，但 UI 文案只说“客户账号”“可查看客户”“客户电脑”。

### 8.3 客户数据包详情展示

客户数据包详情默认展示人类可读摘要：

- 正式知识数量。
- 知识分类数量。
- 商品专属知识文件数量。
- RAG 原始资料、切片、索引数量。
- 包大小、创建时间、备份编号、下载入口。

技术清单折叠展示。技术清单里的字段含义：

- `path`：文件在备份包里的相对路径，用于还原时定位文件。
- `sha256`：文件校验码，用于确认备份文件没有损坏或被篡改。
- `bytes`：文件大小。

这些字段不是加密后的知识正文，也不是管理员日常需要阅读的内容；它们主要服务于备份校验、差异同步和还原流程。若管理员需要看知识正文，应在“客户数据”摘要、Local 客户端知识库或后续的服务端客户数据详情页中查看。

### 8.4 共享公共知识展示

共享公共知识正式库默认以列表 + 详情展示：

- 列表看标题、分类、状态、来源、正文摘要。
- 点开详情看正文、更新时间、编辑和删除入口。
- 不再把原始 JSON 作为主要阅读内容。
- 候选库展示候选标题、来源客户、内容摘要和采纳/拒绝/作废按钮。

### 8.5 客户电脑连接

Local 客户端登录成功后自动调用 VPS 注册接口，把当前电脑注册为“客户电脑连接”。

如果 VPS 页面仍显示暂无客户电脑，优先检查：

- Local 是否配置了 `WECHAT_VPS_BASE_URL`。
- Local 登录拿到的是否是 VPS 认可的 token。
- VPS 服务端是否可访问。
- Local 是否刚重启到包含自动注册逻辑的新版本。

### 8.6 备份删除

VPS 备份记录允许 admin 删除。删除备份记录时：

- 删除服务端备份记录。
- 如果记录关联服务端本地备份文件，也删除对应文件。
- 如果记录关联客户数据包记录，一并清理关联包记录。

### 8.7 客户端更新环境依赖

不建议要求客户手工安装 Git、Python、Node 或其它开发工具。真实可交付安装包应内置或自动配置：

- Python 运行时或打包后的可执行文件。
- 必要 Python 依赖。
- Node/前端构建产物所需运行资源，若运行期不需要 Node，则不安装 Node。
- 更新下载器。
- 包签名/哈希校验。
- 安装前备份。
- 失败回滚。
- Windows 开机自启/服务化配置，按客户需求决定。

如果后续选择“Git 拉取代码更新”的方案，则安装器需要自动安装或内置 Git，并隐藏复杂命令；但更推荐发布 zip/安装包，由 Local 下载、校验、替换、回滚。这个事项应放入后续打包交付任务中实现。
