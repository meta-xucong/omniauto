共享公共知识

Local 只从 customer 正式知识提交候选；管理员采纳后进入云端正式共享库，客户电脑收到通知后拉取云端快照。

云端正式库快照 用于记录云端正式共享公共知识的状态，方便审计和排查；推送命令只通知客户电脑刷新云端快照。
1. 候选待审来自客户 Local 上传
2. 正式共享库admin 确认后的公共知识
3. 生成发布记录每次采纳都会留下 patch 审计记录
4. 手动推送客户电脑轮询后刷新云端快照

客户电脑连接

Local 客户端登录后会向 VPS 报到。这里显示能接收远程备份、还原、检查更新命令的客户电脑。

备份与还原

备份记录可以下载、还原或删除。还原默认只创建演练命令，避免误覆盖客户本地数据。

版本更新

标准做法是先把构建好的更新包放到 VPS 可下载地址，再在这里登记版本。Local 端检查更新时读取这里的发布记录。

登记新版本

标准操作方法 1. 构建 zip/安装包；2. 上传到 VPS 下载目录；3. 在这里填写版本号和下载地址；4. Local 客户端通过“检查更新”读取最新记录。后续打包安装器应内置运行时和更新工具。

账号安全

修改当前 admin 登录密码。保存后建议重新登录，并把新密码同步到安全的密码管理器。

SMTP 与邮箱验证码

服务端统一负责发验证码。客户端和客户账号只需要绑定邮箱，不需要知道 SMTP 密码。

SMTP 状态读取中...
邮箱验证码 正式商用时请在这里配置 SMTP，并确认已勾选“登录和改密必须验证邮箱”。未配置完整 SMTP 时，验证码会写入本机开发 outbox，适合本地模拟测试。
