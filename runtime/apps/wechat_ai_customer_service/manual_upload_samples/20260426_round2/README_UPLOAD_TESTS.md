# 微信 AI 客服资料上传测试样本

这些文件用于手动测试“资料上传 -> AI 学习 -> 候选审核 -> 入库”的流程。内容均为测试造数，不是正式业务资料。

建议上传方式：

| 文件 | 建议门类 | 测试重点 |
| --- | --- | --- |
| `01_products_new_office_devices.xlsx` | 商品资料 | Excel 商品、规格、库存、阶梯价格、物流售后 |
| `02_policy_delivery_invoice.csv` | 政策规则 | CSV 中的发货、开票、对公付款、人工确认规则 |
| `03_chat_records_new_energy_storage.md` | 聊天记录 | 从客服对话中归纳话术风格和可复用回复 |
| `04_erp_export_after_sales.json` | ERP 导出 | 从 ERP 类记录中识别订单、售后、同步状态 |
| `05_mixed_company_faq.txt` | 政策规则 | 普通文本中的公司信息、样品、合同、超界请求 |

验收建议：

1. 先在管理台“资料上传”里逐个上传。
2. 切到“候选审核”，点击“AI 学习”。
3. 查看候选是否能归纳出清晰的结构化知识。
4. 如果候选内容合理，再手动确认入库。
5. 入库后回到“知识库”检查是否按门类保存，且展示为表单/业务字段。
