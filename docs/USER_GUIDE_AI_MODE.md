# OmniAuto AI 模式使用说明书

> **适用场景**: 你不需要写代码，也不需要记住任何 CLI 命令。只需要在 Kimi Code、Codex、Telegram 或微信的聊天窗口里，像跟朋友说话一样给 AI 下达指令，OmniAuto 就会自动完成全部工作。

---

## 1. 快速开始

### 1.1 你现在可以怎么跟 AI 说话？

OmniAuto 支持通过以下入口接收你的指令：

| 入口 | 使用方式 | 适合场景 |
|------|---------|---------|
| **Kimi Code 聊天窗口** | 直接在编辑器侧边栏的 AI 对话里打字 | 开发调试、需要快速迭代脚本 |
| **Codex / OpenClaude** | 在 IDE 的 AI Agent 面板中发送消息 | 深度自动化任务、复杂流程 |
| **Telegram Bot** | 给 Bot 发消息，7×24 小时可用 | 随时随地手机下达指令 |
| **微信** | 给 OmniAuto 机器人发消息 | 国内用户日常办公场景 |

### 1.2 第一次使用

**Kimi Code 示例**：

```
你：帮我访问 httpbin.org，提取页面上的 H1 标题发给我。

AI：好的，我来帮你完成这个任务。
      1. 正在规划步骤...
      2. 正在生成自动化脚本...
      3. 正在执行...
      4. 任务完成！
      
      提取结果：Herman Melville - Moby-Dick
```

全程你不需要写任何代码，AI 会自动调用 OmniAuto 完成：规划 → 生成脚本 → 执行 → 返回结果。

## 3. AI 会帮你做什么？（工作方式说明）

当你发出一条指令后，AI 会自动完成以下闭环，全程不需要你干预：

```
你的自然语言指令
      ↓
[步骤1: 意图理解] AI 判断：这是新任务 / 定时任务 / 查询 / 修复？
      ↓
[步骤2: 任务规划] AI 把任务拆解成 OmniAuto 能执行的原子步骤
      ↓
[步骤3: 脚本生成] AI 自动生成 Python 原子脚本
      ↓
[步骤4: 安全校验] 自动扫描脚本是否有风险操作
      ↓
[步骤5: 预执行确认] 若涉及敏感操作，AI 会先询问你（可配置自动跳过）
      ↓
[步骤6: 自动执行] OmniAuto 状态机严格按照脚本执行每一步
      ↓
[步骤7: 结果观测] 若失败，AI 自动获取截图和错误日志进行分析
      ↓
[步骤8: 自我修复] AI 自动修改脚本并重新执行（最多尝试3次）
      ↓
[步骤9: 结果反馈] 把最终结果、数据、截图返回给你
```

### 3.1 你什么时候需要回应？

**一般情况下**：你发一条指令，等结果就行。

**需要确认时**：AI 会主动问你，例如：
- "这个任务涉及点击登录按钮，是否继续？"
- "检测到需要发送消息到钉钉群，请确认是否执行？"
- "脚本连续3次失败，需要人工介入，这是截图和错误信息..."

### 3.2 执行过程你能看到什么？

AI 会在聊天窗口实时汇报进度：

```
AI：正在规划任务步骤... [1/4]
AI：正在生成自动化脚本... [2/4]
AI：脚本已生成，正在安全校验... [3/4]
AI：正在执行工作流，预计需要 15 秒... [4/4]
AI：✅ 任务完成！提取到 5 条数据：...
```

## 4. 安全与人工确认

### 4.1 哪些操作会被拦截并要求确认？

为了安全，以下操作默认会触发 Guardian（ guardian 节点），AI 会暂停并询问你：

- **发送消息/邮件/通知**（如发到钉钉群、微信群、发邮件）
- **涉及金钱的操作**（如转账、支付、退款）
- **删除/修改重要配置**（如删除数据库记录、修改系统设置）
- **首次访问新的敏感网站**（可配置）

### 4.2 高信任模式

如果你希望减少被打断的次数，可以开启**高信任模式**：

```
你：开启高信任模式，普通浏览和点击操作不用每次问我。
```

开启后，AI 会自动放行低风险操作，但**以下操作永远会要求确认**：
- 发送消息
- 转账/支付
- 删除数据
- 调用系统命令（此功能本身已被永久禁止）

> ⚠️ **安全提示**：高信任模式适合在私密、可控的环境中使用。不要在多人共用设备或不熟悉的网络环境下开启。

## 5. 常见问题

### Q1: 任务卡住/执行时间很长怎么办？

**你**: "刚才那个任务怎么还没完成？"

**AI 会**: 
1. 查询任务状态
2. 如果是网络慢，会等待或增加超时时间
3. 如果是页面卡住，会截图分析并自动重试
4. 如果确实无法继续，会向你汇报原因和截图

### Q2: 我想修改已经创建好的定时任务，怎么办？

**你**: "把每天早上9点打卡的任务改成9点半。"

**AI 会**: 
1. 找到对应的定时任务
2. 删除旧任务
3. 用新的时间重新创建
4. 向你确认修改结果

### Q3: AI 生成的脚本执行错了，我想让它换一种方式，怎么说？

直接告诉 AI 你的要求即可：

```
刚才那个脚本点击按钮时没找到元素，你换成用视觉识别（截图找按钮）的方式再试一次。
```

### Q4: 任务失败后，AI 自动修复了3次还是不行，怎么办？

AI 会把最终的错误信息、失败截图、脚本内容全部发给你。你可以：
- 根据截图告诉 AI "等页面完全加载后再点击"
- 或者暂时放弃，等网站稳定后再试
- 或者切换到手动模式，自己指定具体的 CSS 选择器

### Q5: 我可以同时下发多个任务吗？

可以。例如：

```
帮我做两件事：
1. 现在立刻去淘宝搜机械键盘，前5个商品价格发给我；
2. 每天晚上8点去京东搜同款，如果降价了就通知我。
```

AI 会分别处理这两个任务，一个立即执行，一个创建为定时任务。

### Q6: 我不用 Kimi Code，只用 Telegram/微信可以吗？

完全可以。Telegram Bot 和微信机器人接入后，使用方式和上面的示例完全一致。你在手机上打字，AI 在云端执行，结果推送到你的聊天窗口。

## 6. 高级用法

### 6.1 指定执行参数

如果你想对执行过程做更精细的控制，可以在指令里说明：

```
访问这个页面时不要用无头模式，我想看到浏览器窗口。
```
（AI 会调用 `headless=false` 执行）

```
这个任务很敏感，执行前一定要先截图给我看看当前页面状态。
```
（AI 会在关键步骤前插入截图步骤）

### 6.2 让 AI 分析历史结果

```
把这一周每天早上9点采集的股价数据汇总一下，告诉我哪天涨幅最大。
```

AI 会：
1. 从 StateStore 中查询这一周所有相关任务的输出数据
2. 读取并进行汇总分析
3. 返回结论

### 6.3 让 AI 学习你的偏好

```
以后所有涉及登录的操作，都默认等待3秒再输入密码。
```

AI 会将这个偏好记录到你的用户配置文件中，后续生成脚本时自动应用。
